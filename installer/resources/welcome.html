mSL/XNU presents macOS through a Linux-shaped namespace: home directories under /home, removable volumes under /media, and /mnt for manual mounts.

This is not a container, a virtual machine, or an emulator. There is no second kernel and no translation layer — the processes are macOS processes and the kernel is XNU. Only the shape of the namespace changes. No kernel extension is installed and no security settings are altered.

This installer places the tools, a background daemon, a menu-bar app and a preference pane. Nothing is switched on. Turning a component on changes system configuration, so that decision is left to you — use the mSL/XNU pane in System Settings once the install finishes.

A restart is needed for /mnt and /media. macOS creates root-level directories only at startup, so those two appear after you restart. /home applies immediately.
