mSL/XNU is installed, and nothing is enabled yet.

Open System Settings → mSL/XNU, or use the menu-bar icon, and switch on the components you want. Flip as many as you like and press Apply: they are applied together, behind one authorization.

/home takes effect at once. /mnt and /media appear after a restart, because macOS creates root-level directories only at startup.

From the command line, mslctl status shows the state of everything and mslctl home check reports whether /home is safe to enable on this machine.

To remove mSL/XNU, run Uninstall mSL.command from the disk image. It switches every component off before removing anything, so the system is left as it was found.
